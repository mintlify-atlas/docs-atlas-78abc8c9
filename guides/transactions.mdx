## Overview

Transactions are the core of Expensia Finance Tracker. This guide covers creating, retrieving, updating, and deleting transactions, along with filtering and balance management.

## Transaction Types

Expensia supports two transaction types:

- `income` - Money received
- `expense` - Money spent

## Transaction Object

<ResponseField name="id" type="integer">
  Unique identifier for the transaction
</ResponseField>

<ResponseField name="transaction_type" type="string" required>
  Type of transaction: `income` or `expense`
</ResponseField>

<ResponseField name="amount" type="float" required>
  Transaction amount (positive number)
</ResponseField>

<ResponseField name="note" type="string">
  Optional note or description for the transaction
</ResponseField>

<ResponseField name="category_id" type="integer">
  Optional category ID to associate with this transaction
</ResponseField>

<ResponseField name="transaction_date" type="date">
  Date of the transaction (YYYY-MM-DD format). Defaults to current date if not provided
</ResponseField>

<ResponseField name="created_date" type="datetime">
  Timestamp when the transaction was created (read-only)
</ResponseField>

<ResponseField name="updated_date" type="datetime">
  Timestamp when the transaction was last updated (read-only)
</ResponseField>

## Creating Transactions

### Basic Transaction

<CodeGroup>
```bash cURL
curl -X POST "https://api.expensia.com/transactions" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "transaction_type": "expense",
    "amount": 50.00,
    "note": "Lunch at restaurant"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.expensia.com/transactions",
    headers={"Authorization": "Bearer YOUR_TOKEN"},
    json={
        "transaction_type": "expense",
        "amount": 50.00,
        "note": "Lunch at restaurant"
    }
)

print(response.json())
```

```javascript JavaScript
const response = await fetch('https://api.expensia.com/transactions', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    transaction_type: 'expense',
    amount: 50.00,
    note: 'Lunch at restaurant'
  })
});

const data = await response.json();
console.log(data);
```
</CodeGroup>

**Response (201 Created):**
```json
{
  "transaction_id": 123,
  "current_balance": 950.00
}
```

<Info>
The API automatically updates your account balance when creating transactions:
- **Expenses** decrease the balance
- **Income** increases the balance
</Info>

## Retrieving Transactions

Retrieve all transactions for the authenticated user, ordered by date (newest first).

<CodeGroup>

```python Python
transactions = response.json()["transactions"]
```

## Updating Transactions

Update an existing transaction. All fields in the request body will replace the existing values.

<CodeGroup>
```bash cURL
curl -X PUT "https://api.expensia.com/transactions/123" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "transaction_type": "expense",
    "amount": 65.00,
    "note": "Lunch at restaurant (updated amount)",
    "category_id": 2,
    "transaction_date": "2024-01-15"
  }'
```

```python Python
response = requests.put(
    "https://api.expensia.com/transactions/123",
    headers={"Authorization": "Bearer YOUR_TOKEN"},
    json={
        "transaction_type": "expense",
        "amount": 65.00,
        "note": "Lunch at restaurant (updated amount)",
        "category_id": 2,
        "transaction_date": "2024-01-15"
    }
)
```
</CodeGroup>

**Response (200 OK):**
```json
{
  "transaction_id": 123,
  "message": "Transaction updated successfully",
  "current_balance": 935.00
}
```

<Accordion title="How balance is recalculated on update">
When updating a transaction, the API:

1. **Rolls back** the old transaction's effect on balance
2. **Applies** the new transaction's effect

For example:
- Old transaction: expense of $50 (balance was reduced by $50)
- New transaction: expense of $65
- Balance change: +$50 (rollback) - $65 (new) = -$15 net change

This is handled automatically by the `adjust_user_balance` function in `/home/daytona/workspace/source/utils/utils.py:70`.
</Accordion>

## Deleting Transactions

### Delete Single Transaction

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.expensia.com/transactions/123" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```python Python
response = requests.delete(
    "https://api.expensia.com/transactions/123",
    headers={"Authorization": "Bearer YOUR_TOKEN"}
)
```
</CodeGroup>

**Response (200 OK):**
```json
{
  "deleted_transaction_id": 123,
  "message": "Transaction deleted"
}
```

<Info>
When a transaction is deleted:
- The balance is automatically adjusted (reversed)
- The user's `total_transactions` count is decremented
- If the transaction was an expense, `total_expenses` is decreased
- If the transaction was income, `total_income` is decreased
</Info>

### Delete All Transactions

<Warning>
This action is **irreversible** and will delete all transactions for the authenticated user.
</Warning>

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.expensia.com/transactions" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```python Python
response = requests.delete(
    "https://api.expensia.com/transactions",
    headers={"Authorization": "Bearer YOUR_TOKEN"}
)
```
</CodeGroup>

**Response (200 OK):**
```json
{
  "message": "All transactions deleted"
}
```

<Note>
When all transactions are deleted, the following user fields are reset to zero:
- `current_balance`
- `total_expenses`
- `total_income`
- `total_transactions`
</Note>

## Best Practices

<AccordionGroup>
  <Accordion title="Always provide meaningful notes">
    Including descriptive notes helps with tracking and categorizing expenses later. Even simple notes like "Grocery shopping" or "Coffee" are better than no note.
  </Accordion>

  <Accordion title="Use categories consistently">
    Assign categories to transactions to enable better filtering and analysis. Create categories before creating transactions that reference them.
  </Accordion>

  <Accordion title="Specify transaction dates">
    While `transaction_date` is optional and defaults to the current date, explicitly setting it ensures accurate historical tracking, especially when entering past transactions.
  </Accordion>

  <Accordion title="Handle errors gracefully">
    Always check for error responses and handle them appropriately. Common errors include invalid category IDs (404) and validation errors (400).
  </Accordion>
</AccordionGroup>
