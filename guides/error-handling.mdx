## Overview

The Expensia API uses standard HTTP status codes and returns detailed error messages in JSON format. This guide covers common errors you'll encounter and how to handle them effectively.

## Error Response Format

All error responses follow this structure:

```json
{
  "detail": "Error message describing what went wrong"
}
```

---

## Authentication Errors

### 401 Unauthorized - Token Expired

**Error Response:**
```json
{
  "detail": "Token expired"
}
```

**Cause:**
Your access token has expired. Access tokens are valid for a limited time (configured by `ACCESS_TOKEN_EXPIRE_MINUTES`).

**Solution:**
Request a new access token using the refresh token endpoint or log in again.

<CodeGroup>
```python Python
import requests

def handle_token_expiry(error_response):
    if error_response.get("detail") == "Token expired":
        # Option 1: Use refresh token
        refresh_response = requests.post(
            "https://api.expensia.com/auth/refresh",
            headers={"Authorization": f"Bearer {refresh_token}"}
        )
        new_token = refresh_response.json()["access_token"]
        
        # Retry the original request with new token
        return new_token
    
    return None
```

```javascript JavaScript
async function handleTokenExpiry(error) {
  if (error.detail === 'Token expired') {
    // Request new token
    const response = await fetch('https://api.expensia.com/auth/refresh', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${refreshToken}`
      }
    });
    
    const data = await response.json();
    return data.access_token;
  }
  return null;
}
```
</CodeGroup>

<Note>
**Implementation reference:** The token expiry check is implemented in `utils/utils.py:52-56`:

```python
except jwt.ExpiredSignatureError:

        status_code=status.HTTP_401_UNAUTHORIZED,

```
</Note>

## Resource Not Found Errors

### 404 Not Found - Transaction Not Found

**Error Response:**
```json
{
  "detail": "Transaction not found"
}
```

**Cause:**
- The transaction ID doesn't exist
- The transaction belongs to a different user
- The transaction was already deleted

**Where it occurs:**
- `GET /transactions/{id}` - When retrieving a single transaction
- `PUT /transactions/{id}` - When updating a transaction
- `DELETE /transactions/{id}` - When deleting a transaction

<CodeGroup>
```python Python
try:
    response = requests.get(
        f"https://api.expensia.com/transactions/{transaction_id}",
        headers={"Authorization": f"Bearer {token}"}
    )
    response.raise_for_status()
except requests.exceptions.HTTPError as e:
    if e.response.status_code == 404:
        print("Transaction not found. It may have been deleted or doesn't exist.")
    else:
        raise
```

```javascript JavaScript
try {
  const response = await fetch(
    `https://api.expensia.com/transactions/${transactionId}`,
    { headers: { 'Authorization': `Bearer ${token}` } }
  );
  
  if (response.status === 404) {
    console.log('Transaction not found');
    return null;
  }
  
  return await response.json();
} catch (error) {
  console.error('Request failed:', error);
}
```
</CodeGroup>

**Implementation:**

### 404 Not Found - Category Not Found

**Error Response:**
```json
{
  "detail": "Category not found"
}
```

**Cause:**
- The category ID doesn't exist
- The category belongs to a different user
- Attempting to assign a non-existent category to a transaction

**Where it occurs:**
- `GET /category/{id}` - When retrieving a single category
- `GET /category/{id}/transactions` - When getting category transactions
- `POST /transactions` - When creating a transaction with invalid category_id
- `PUT /transactions/{id}` - When updating a transaction with invalid category_id
- `DELETE /category/{id}` - When deleting a category

**Implementation:**


## Validation Errors

### 400 Bad Request - Invalid Date Parameters

#### Negative Timestamp

**Error Response:**
```json
{
  "detail": "start_date_ms must be positive"
}
```

or

```json
{
  "detail": "end_date_ms must be positive"
}
```

**Cause:**
Date parameters must be positive integers (milliseconds since Unix epoch).

**Implementation:**
```python
# From routes/transactionRoutes.py:35-38
if start_date_ms is not None and start_date_ms < 0:
    raise HTTPException(status_code=400, detail="start_date_ms must be positive")
if end_date_ms is not None and end_date_ms < 0:
    raise HTTPException(status_code=400, detail="end_date_ms must be positive")
```

**Solution:**
```python
# Correct: Positive timestamp
from datetime import datetime
start_ms = int(datetime(2024, 1, 1).timestamp() * 1000)  # Positive value

# Incorrect: Negative value
start_ms = -1000  # Will fail
```

#### Invalid Date Range

**Error Response:**
```json
{
  "detail": "start_date cannot be greater than end_date"
}
```

**Cause:**
The start date is after the end date.

**Implementation:**
```python
# From routes/transactionRoutes.py:46-49
if start_date and end_date and start_date > end_date:
    raise HTTPException(
        status_code=400, detail="start_date cannot be greater than end_date"
    )
```

**Solution:**
<CodeGroup>
```python Python
from datetime import datetime

start_date = datetime(2024, 1, 1)
end_date = datetime(2024, 1, 31)

# Validate before making request
if start_date > end_date:
    raise ValueError("Start date must be before end date")

start_ms = int(start_date.timestamp() * 1000)
end_ms = int(end_date.timestamp() * 1000)

response = requests.get(
    "https://api.expensia.com/transactions",
    params={"start_date_ms": start_ms, "end_date_ms": end_ms},
    headers={"Authorization": f"Bearer {token}"}
)
```

```javascript JavaScript
const startDate = new Date('2024-01-01');
const endDate = new Date('2024-01-31');

// Validate before making request
if (startDate > endDate) {
  throw new Error('Start date must be before end date');
}

const startMs = startDate.getTime();
const endMs = endDate.getTime();

const response = await fetch(
  `https://api.expensia.com/transactions?start_date_ms=${startMs}&end_date_ms=${endMs}`,
  { headers: { 'Authorization': `Bearer ${token}` } }
);
```
</CodeGroup>

## Conflict Errors

### 409 Conflict - Duplicate Category Name

**Error Response:**
```json
{
  "detail": "Category name already exists"
}
```

**Cause:**
Attempting to create a category with a name that already exists for the authenticated user.

**Implementation:**
```python
# From routes/categoryRoutes.py:78-83
existing = db.query(models.Category).filter(
    models.Category.user_id == currentUser.id,
    models.Category.name == category.name
).first()
if existing:
    raise HTTPException(status_code=409, detail="Category name already exists")
```

**Solution:**
<CodeGroup>
```python Python
def create_category_safe(name, color, icon, token):
    # First, check if category already exists
    response = requests.get(
        "https://api.expensia.com/categories",
        headers={"Authorization": f"Bearer {token}"}
    )
    
    categories = response.json()["categories"]
    existing_names = [cat["name"] for cat in categories]
    
    if name in existing_names:
        print(f"Category '{name}' already exists. Using existing category.")
        existing_cat = next(cat for cat in categories if cat["name"] == name)
        return existing_cat["id"]
    
    # Create new category
    response = requests.post(
        "https://api.expensia.com/categories",
        headers={"Authorization": f"Bearer {token}"},
        json={"name": name, "color": color, "icon": icon}
    )
    
    if response.status_code == 201:
        return response.json()["id"]
    elif response.status_code == 409:
        # Race condition: category was created between check and create
        print(f"Category '{name}' was just created. Fetching ID...")
        # Fetch categories again to get the ID
        response = requests.get(
            "https://api.expensia.com/categories",
            headers={"Authorization": f"Bearer {token}"}
        )
        categories = response.json()["categories"]
        existing_cat = next(cat for cat in categories if cat["name"] == name)
        return existing_cat["id"]
```

```javascript JavaScript
async function createCategorySafe(name, color, icon, token) {
  try {
    const response = await fetch('https://api.expensia.com/categories', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ name, color, icon })
    });
    
    if (response.status === 201) {
      const data = await response.json();
      return data.id;
    } else if (response.status === 409) {
      console.log(`Category '${name}' already exists`);
      
      // Fetch existing category ID
      const categoriesResponse = await fetch(
        'https://api.expensia.com/categories',
        { headers: { 'Authorization': `Bearer ${token}` } }
      );
      
      const { categories } = await categoriesResponse.json();
      const existing = categories.find(cat => cat.name === name);
      return existing.id;
    }
  } catch (error) {
    console.error('Failed to create category:', error);
    throw error;
  }
}
```
</CodeGroup>

## Database & Server Errors

### 500 Internal Server Error - Database Commit Failed

While not explicitly shown in error responses, database operations use try-catch blocks with rollback:

```python
# From routes/transactionRoutes.py:159-163
try:
    db.commit()
except:
    db.rollback()
    raise
```

**Cause:**
- Database connection issues
- Constraint violations
- Data integrity errors

**Solution:**
<CodeGroup>
```python Python
import time
import requests
from requests.exceptions import HTTPError

def create_transaction_with_retry(transaction_data, token, max_retries=3):
    for attempt in range(max_retries):
        try:
            response = requests.post(
                "https://api.expensia.com/transactions",
                headers={"Authorization": f"Bearer {token}"},
                json=transaction_data
            )
            response.raise_for_status()
            return response.json()
        except HTTPError as e:
            if e.response.status_code >= 500:
                if attempt < max_retries - 1:
                    wait_time = 2 ** attempt  # Exponential backoff
                    print(f"Server error. Retrying in {wait_time}s...")
                    time.sleep(wait_time)
                else:
                    print("Max retries reached. Server error persists.")
                    raise
            else:
                # Client error (4xx) - don't retry
                raise
```
</CodeGroup>

  <Accordion title="Always check status codes">
    ```python

        "https://api.expensia.com/transactions",
        headers={"Authorization": f"Bearer {token}"},
        json=transaction_data
    )
    
    if response.status_code == 201:

        return response.json()
    elif response.status_code == 404:
        print("Category not found")
    ```

    ```python
    from requests.packages.urllib3.util.retry import Retry
    
    # Configure automatic retries for 5xx errors
    retry_strategy = Retry(
        total=3,
        status_forcelist=[500, 502, 503, 504],
        backoff_factor=1  # 1s, 2s, 4s
    )
    
    adapter = HTTPAdapter(max_retries=retry_strategy)
    session = requests.Session()
    session.mount("https://", adapter)
    
    # Use session for requests
    response = session.post(
        "https://api.expensia.com/transactions",
        headers={"Authorization": f"Bearer {token}"},

    )
    ```

## Error Code Reference

<ResponseField name="200" type="OK">
  Request succeeded. Resource retrieved or operation completed successfully.
</ResponseField>

<ResponseField name="201" type="Created">
  Resource created successfully (transactions, categories).
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request parameters or validation failure.
  - Negative date timestamps
  - Invalid date range (start > end)
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Authentication failed or token invalid/expired.
  - Token expired
  - Invalid token
</ResponseField>

<ResponseField name="404" type="Not Found">
  Requested resource doesn't exist or doesn't belong to user.
  - Transaction not found
  - Category not found
</ResponseField>

<ResponseField name="409" type="Conflict">
  Resource already exists.
  - Duplicate category name
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server-side error occurred. Retry the request or contact support if it persists.
</ResponseField>
