## Overview

The Expensia API supports filtering transactions by date range using millisecond timestamps. This guide explains how to use the date filtering parameters and handle timezone conversions.

## Query Parameters

The `/transactions` endpoint accepts optional date filtering parameters:

<ParamField query="start_date_ms" type="integer" optional>
  Start date in milliseconds since Unix epoch (must be positive)
</ParamField>

<ParamField query="end_date_ms" type="integer" optional>
  End date in milliseconds since Unix epoch (must be positive)
</ParamField>

<Info>
**How date filtering works:**
- Dates are converted from milliseconds to UTC datetime
- Start date is set to 00:00:00 (beginning of day)
- End date is set to 23:59:59 (end of day, by adding 1 day and filtering with `<`)
- Transactions are filtered inclusively for both start and end dates
- Results are ordered by transaction date (newest first)
</Info>

## Basic Date Filtering

### Filter by Date Range

<CodeGroup>
```bash cURL
# January 1, 2024 to January 31, 2024
curl -X GET "https://api.expensia.com/transactions?start_date_ms=1704067200000&end_date_ms=1706745599000" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```python Python
import requests
from datetime import datetime

# Convert dates to milliseconds
start_date = datetime(2024, 1, 1)
end_date = datetime(2024, 1, 31)

start_ms = int(start_date.timestamp() * 1000)
end_ms = int(end_date.timestamp() * 1000)

response = requests.get(
    "https://api.expensia.com/transactions",
    headers={"Authorization": "Bearer YOUR_TOKEN"},
    params={
        "start_date_ms": start_ms,
        "end_date_ms": end_ms
    }
)

data = response.json()
print(f"Found {len(data['transactions'])} transactions")
```

```javascript JavaScript
// January 1, 2024 to January 31, 2024
const startDate = new Date('2024-01-01');
const endDate = new Date('2024-01-31');

const startMs = startDate.getTime();
const endMs = endDate.getTime();

const response = await fetch(
  `https://api.expensia.com/transactions?start_date_ms=${startMs}&end_date_ms=${endMs}`,
  {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);

const data = await response.json();
console.log(`Found ${data.transactions.length} transactions`);
```
</CodeGroup>

### No Date Filters

When no date parameters are provided, all transactions are returned.

<CodeGroup>
```bash cURL
curl -X GET "https://api.expensia.com/transactions" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

**Response (200 OK):**
```json
{
  "transactions": [
    // All transactions for the user
  ]
}
```

<Note>
When no date filters are provided, the response **does not** include `start_date_ms` or `end_date_ms` fields.
</Note>

## Date Conversion Implementation

The API uses the `ms_to_utc_nepal` utility function to convert milliseconds to datetime:

```python
# From utils/utils.py:64-67
def ms_to_utc_nepal(ms: int) -> datetime:
    nepal_time = datetime.fromtimestamp(ms / 1000)
    utc_time = nepal_time.replace(microsecond=0, tzinfo=timezone.utc)
    return utc_time
```

### Date Filtering Logic

The filtering logic in the API:

```python
# From routes/transactionRoutes.py:40-61
start_date = (
    utils.ms_to_utc_nepal(start_date_ms) if start_date_ms is not None else None
)
end_date = utils.ms_to_utc_nepal(end_date_ms) if end_date_ms is not None else None

if start_date:
    start_date = start_date.replace(hour=0, minute=0, second=0)
    query = query.filter(models.Transaction.transaction_date >= start_date)

if end_date:
    end_date = end_date.replace(hour=0, minute=0, second=0)
    end_date += timedelta(days=1)  # Include entire end day
    query = query.filter(models.Transaction.transaction_date < end_date)
```

<Accordion title="Why add 1 day to end_date?">
The end date is increased by 1 day and uses the `<` (less than) operator instead of `<=` (less than or equal). This ensures that transactions with any time on the end date are included.

**Example:**
- User requests: `end_date_ms` = January 31, 2024 at 10:30 AM
- API converts to: January 31, 2024 at 00:00:00
- API adds 1 day: February 1, 2024 at 00:00:00
- Query filters: `transaction_date < February 1, 2024 00:00:00`
- Result: All transactions on January 31, 2024 (any time) are included
</Accordion>

## Validation Rules

### Positive Values Only

Both date parameters must be positive integers.

<CodeGroup>
```bash cURL - Invalid Request
curl -X GET "https://api.expensia.com/transactions?start_date_ms=-1000" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

**Error Response (400 Bad Request):**
```json
{
  "detail": "start_date_ms must be positive"
}
```

```python
# From routes/transactionRoutes.py:35-38
if start_date_ms is not None and start_date_ms < 0:
    raise HTTPException(status_code=400, detail="start_date_ms must be positive")
if end_date_ms is not None and end_date_ms < 0:
    raise HTTPException(status_code=400, detail="end_date_ms must be positive")
```

### Start Date Before End Date

The start date must not be greater than the end date.

<CodeGroup>
```bash cURL - Invalid Request
# Start: Feb 1, 2024, End: Jan 1, 2024 (invalid)
curl -X GET "https://api.expensia.com/transactions?start_date_ms=1706745600000&end_date_ms=1704067200000" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

**Error Response (400 Bad Request):**
```json
{
  "detail": "start_date cannot be greater than end_date"
}
```

```python
# From routes/transactionRoutes.py:46-49
if start_date and end_date and start_date > end_date:
    raise HTTPException(
        status_code=400, detail="start_date cannot be greater than end_date"
    )
```

## Timezone Considerations

<Warning>
**Important:** The API converts timestamps to UTC timezone. If you're working with dates in a different timezone, ensure you convert them appropriately before sending to the API.
</Warning>

### Example: Converting Local Time to UTC

<CodeGroup>
```python Python
from datetime import datetime
import pytz

# Create date in local timezone (e.g., US Eastern)
local_tz = pytz.timezone('US/Eastern')
local_date = local_tz.localize(datetime(2024, 1, 15, 10, 30))

# Convert to milliseconds (automatically handles UTC conversion)
ms = int(local_date.timestamp() * 1000)

print(f"Milliseconds: {ms}")
```

```javascript JavaScript
// JavaScript Date objects automatically handle timezone conversion
const localDate = new Date('2024-01-15T10:30:00'); // Local timezone
const ms = localDate.getTime(); // Automatically converts to UTC

console.log(`Milliseconds: ${ms}`);
```
</CodeGroup>
