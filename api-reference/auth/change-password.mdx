---
title: Change Password
api: PATCH /change-password
description: Change the authenticated user's password
---

<Note>
  This endpoint requires authentication. Include a valid access token in the Authorization header.
</Note>

## Authentication

This endpoint requires a Bearer token in the Authorization header:

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Request Body

<ParamField body="current_password" type="string" required>
  The user's current password for verification
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password to set. Will be hashed before storage.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming password change
</ResponseField>

<ResponseField name="access_token" type="string">
  New JWT access token for continued authentication after password change
</ResponseField>

<RequestExample>

```bash cURL
curl -X PATCH https://api.expensia.com/change-password \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "current_password": "oldPassword123",
    "new_password": "newSecurePassword456"
  }'
```

```json Request
{
  "current_password": "oldPassword123",
  "new_password": "newSecurePassword456"
}
```

</RequestExample>

<ResponseExample>

```json 200 - Success
{
  "message": "Password changed successfully",
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

```json 401 - Incorrect Current Password
{
  "detail": "Current password is incorrect"
}
```

```json 401 - Unauthorized
{
  "detail": "Not authenticated"
}
```

</ResponseExample>

## Notes

- Current password must match the stored password
- New password is automatically hashed using secure hashing algorithms
- A new access token is issued after successful password change
- Use the new access token for subsequent requests
- Returns HTTP 200 on success
- Returns HTTP 401 if the current password is incorrect or authentication fails
