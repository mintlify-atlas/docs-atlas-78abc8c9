---
title: List All Users
api: GET /users
description: Retrieve a list of all users in the system
---

<Warning>
  This endpoint does not require authentication and returns all users. It should only be used in development/testing environments and should be disabled in production.
</Warning>

## Response

Returns a list of all registered users with their profile information and financial summaries.

<ResponseField name="users" type="array">
  Array of user objects

  <Expandable title="User object">
    <ResponseField name="id" type="integer">
      Unique user identifier
    </ResponseField>

    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>

    <ResponseField name="name" type="string">
      User's display name
    </ResponseField>

    <ResponseField name="current_balance" type="number">
      Current balance (total income - total expenses)
    </ResponseField>

    <ResponseField name="total_transactions" type="integer">
      Total number of transactions
    </ResponseField>

    <ResponseField name="total_expenses" type="number">
      Sum of all expense transactions
    </ResponseField>

    <ResponseField name="total_income" type="number">
      Sum of all income transactions
    </ResponseField>
  </Expandable>
</ResponseField>

## Request Example

<CodeGroup>
```bash cURL
curl -X GET https://api.expensia.com/users
```

```python Python
import requests

response = requests.get("https://api.expensia.com/users")
users = response.json()

print(f"Total users: {len(users['users'])}")
```

```javascript JavaScript
const response = await fetch('https://api.expensia.com/users');
const data = await response.json();

console.log(`Total users: ${data.users.length}`);
```
</CodeGroup>

## Security Considerations

<Warning>
  **Important**: This endpoint exposes all user data without authentication. It should be:
  - Disabled or removed in production environments
  - Protected with admin-level authentication
  - Used only for development and testing purposes
</Warning>

Consider implementing proper access control:
- Add authentication middleware with admin role verification
- Remove the endpoint entirely for production deployments
- Use environment-based feature flags to disable in production
