---
title: Get Current User
api: GET /me
description: Retrieve the authenticated user's profile information
---

<Note>
  This endpoint requires authentication. Include a valid access token in the Authorization header.
</Note>

## Authentication

This endpoint requires a Bearer token in the Authorization header:

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Response

<ResponseField name="id" type="integer">
  The unique identifier of the user
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address
</ResponseField>

<ResponseField name="name" type="string">
  The user's full name
</ResponseField>

<ResponseField name="current_balance" type="float">
  The user's current balance calculated from all transactions
</ResponseField>

<ResponseField name="total_transactions" type="integer">
  Total number of transactions for this user
</ResponseField>

<ResponseField name="total_expenses" type="float">
  Sum of all expense transactions
</ResponseField>

<ResponseField name="total_income" type="float">
  Sum of all income transactions
</ResponseField>

<RequestExample>

```bash cURL
curl -X GET https://api.expensia.com/me \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

</RequestExample>

<ResponseExample>

```json 200 - Success
{
  "id": 42,
  "email": "john.doe@example.com",
  "name": "John Doe",
  "current_balance": 5420.50,
  "total_transactions": 156,
  "total_expenses": 3250.75,
  "total_income": 8671.25
}
```

```json 401 - Unauthorized
{
  "detail": "Not authenticated"
}
```

```json 401 - Invalid Token
{
  "detail": "Could not validate credentials"
}
```

</ResponseExample>

## Notes

- The user is identified from the access token
- Financial summary fields (balances, totals) are computed from the user's transaction history
- Returns HTTP 200 on success
- Returns HTTP 401 if the access token is missing, invalid, or expired
