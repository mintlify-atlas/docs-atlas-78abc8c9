---
title: "Get Recent Transactions"
api: "GET /transactions/recent"
description: "Retrieve transactions from the last 3 days"
---

Fetches all transactions for the authenticated user from the last 3 days, ordered by transaction date (newest first). This endpoint is useful for displaying recent activity on dashboards or home screens.

## Authentication

Requires a valid Bearer token in the Authorization header.

## Response

<ResponseField name="message" type="string">
  Status message indicating the result
  
  **Possible values:**
  - `Recent transactions retrieved successfully` - When transactions are found
  - `No recent transactions found` - When no transactions exist in the last 3 days
</ResponseField>

<ResponseField name="transactions" type="array">
  Array of transaction objects from the last 3 days
  
  <ResponseField name="id" type="integer">
    Unique transaction identifier
  </ResponseField>
  
  <ResponseField name="transaction_type" type="string">
    Type of transaction. Enum: `income`, `expense`
  </ResponseField>
  
  <ResponseField name="amount" type="float">
    Transaction amount
  </ResponseField>
  
  <ResponseField name="note" type="string" optional>
    Optional note or description for the transaction
  </ResponseField>
  
  <ResponseField name="category_id" type="integer" optional>
    ID of the associated category (if assigned)
  </ResponseField>
  
  <ResponseField name="transaction_date" type="datetime" optional>
    Date when the transaction occurred
  </ResponseField>
  
  <ResponseField name="created_date" type="datetime">
    Timestamp when the transaction was created
  </ResponseField>
  
  <ResponseField name="updated_date" type="datetime">
    Timestamp when the transaction was last updated
  </ResponseField>
</ResponseField>

## Error Responses

<ResponseField name="401" type="error">
  Unauthorized - Invalid or missing authentication token
</ResponseField>

## Notes

- The "last 3 days" is calculated from the current UTC time
- Transactions are ordered by `transaction_date` in descending order (newest first)
- An empty array is returned if no transactions exist in the time range
- Only transactions belonging to the authenticated user are returned
- This endpoint does not accept any query parameters - the 3-day window is fixed
