---
title: "Delete All Transactions"
api: "DELETE /transactions"
description: "Delete all transactions for the authenticated user"
---

Deletes all transactions belonging to the authenticated user and resets all financial counters to zero:

- `current_balance` → 0
- `total_income` → 0
- `total_expenses` → 0
- `total_transactions` → 0

This endpoint is useful for resetting a user's financial data or clearing test data.

## Authentication

Requires a valid Bearer token in the Authorization header.

## Reset Behavior

This endpoint performs the following operations:

1. **Deletes all transactions** where `user_id` matches the authenticated user
2. **Resets user financial fields:**
   - `current_balance = 0`
   - `total_expenses = 0`
   - `total_income = 0`
   - `total_transactions = 0`

All operations are performed within a single database transaction to ensure data consistency.

## Notes

- This operation is **irreversible** - all transactions are permanently deleted
- Only transactions belonging to the authenticated user are deleted
- All changes are performed atomically within a database transaction
- If the database commit fails, all changes are rolled back
- This endpoint does not delete categories - only transactions
- Consider implementing a confirmation step in your application before calling this endpoint
