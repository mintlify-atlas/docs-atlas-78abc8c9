---
title: "Delete Transaction"
api: "DELETE /transactions/{id}"
description: "Delete a single transaction by ID"
---

Deletes a specific transaction for the authenticated user. The user's balance is automatically updated to reverse the transaction:

- **Income transaction**: Subtracts from `current_balance` and `total_income`
- **Expense transaction**: Adds back to `current_balance` and subtracts from `total_expenses`

The user's `total_transactions` counter is also decremented.

## Authentication

Requires a valid Bearer token in the Authorization header.

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the transaction to delete
</ParamField>

## Response

<ResponseField name="deleted_transaction_id" type="integer">
  The ID of the deleted transaction
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message: `Transaction deleted`
</ResponseField>

</ResponseField>

## Balance Adjustment

When a transaction is deleted, the following user fields are updated:

### If deleted transaction was Income:
- `current_balance -= amount`
- `total_income -= amount`

### If deleted transaction was Expense:
- `current_balance += amount`
- `total_expenses -= amount`

### Always:
- `total_transactions -= 1`

This effectively reverses the transaction's impact on the user's financial records.

## Notes

- The transaction must belong to the authenticated user
- All balance updates are performed atomically within a database transaction
- If the database commit fails, all changes are rolled back and the transaction is not deleted
- This operation cannot be undone - the transaction is permanently deleted from the database
